## 一、系統定位

不是 ERP Platform、不是 Dashboard，是 **Autonomous Supply Chain Operating System**（自主式供應鏈作戰系統）。

> **BOSS 只想知道兩件事**：
> 1. **哪裡快爆了？**
> 2. **該先救哪裡？**

系統用 **Military Command Center**（軍事作戰指揮中心）思維設計，每秒回答 BOSS 這 2 個問題。

---

## 二、5-Layer 軍事架構

| Layer | 名稱 | 角色 | URL |
|---|---|---|---|
| **L1** | Global Command Center 軍事指揮中心 | CEO / VP / 採購總監 — 3 秒看公司是否安全 | `/erp` |
| **L2** | Operational Centers 六大作戰中心 | 領域 owner — 各自指揮 | `/os/{center}` |
| **L3** | Expert Workbench 專業工作台（32 個）| 操作執行者 — 聚焦單一工作流 | `/os/{center}/{tool}` |
| **L4** | AI Engine Layer 系統大腦（4 大引擎）| Event / Twin / Prediction / Time | `/erp/admin/engines` |
| **L5** | Data / Event / Governance Layer | 底層治理（MDM/RBAC/Audit）| `/erp/admin/*` |

完整架構圖：`/architecture`

## 八、關鍵工作流程 SOP

### A. 早上 8:30 — 副總開機
1. 開 `/erp` 看 banner 顏色（紅/橘/綠）
2. 紅燈 → 看 Triage Queue 第一個救援項
3. 點進去 → 看 AI 建議的方案
4. 拍板 → 進「決策閉環中心」/erp/decisions
5. 5 階段自動執行 + 績效記錄

### B. 採購處理 PO 確認
1. `/os/supplier/portal` 看待處理
2. PO 48hr 未確認 → 系統自動 push 採購主管
3. 點 PO 進入 `/erp/supplier-portal` → 看完整 4 模組
4. 透過供應商提交入口 `/erp/supplier-portal/vendor` 給供應商填

### C. 倉管收貨 7 階段
1. `/os/inventory/receiving` 進入收貨清單
2. 選擇 PO → 進入 7 階段風控
3. Step 1-6 逐一檢核（任何 FAIL → HOLD 連鎖動作）
4. Step 7 自動判定 PASS → 入庫；FAIL → 通知採購+SQE

### D. 議價會議
1. `/os/procurement/should-cost` 輸入對方喊漲 %
2. AI 算合理區間（依各成分波動）
3. 列印議價拿話清單
4. 會議當場使用 → 重新議價

### E. 收到客戶急單
1. `/os/decision/simulation` 進入訂單衝擊模擬器
2. 輸入機種 / 數量 / 新交期
3. AI 30 秒推 3 方案（加急 / 延後低毛利 / 回客戶最早日）
4. 拍板 → 5 階段閉環自動執行

## 十、與鼎新 ERP 整合（IT 部署）

### 整合三層級
- **方式 C** 手動上傳（馬上能用，0 IT 投資）
- **方式 B** 排程匯出資料夾 — 用鼎新原廠 `SQLTOEXCEL.exe`（推薦！）
- **方式 A** Linked Server 唯讀（即時，2-3 天 DBA 工作）

詳見：`/erp/integration`

### 重要原則
- **對鼎新 ERP 唯讀不回寫**
- **不呼叫任何鼎新 Stored Procedure**
- **不 INSERT/UPDATE/DELETE 鼎新任何資料表**
- **扣帳一律回 ERP 操作**
